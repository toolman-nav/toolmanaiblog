# H5 Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a site-wide dark mode that follows the system preference by default and provides a persistent header toggle.

**Architecture:** Theme selection is controlled by `data-theme` on `<html>`. `SiteLayout.astro` sets the first theme before CSS loads, `site.js` owns interaction and persistence, and `styles.css` maps all interface colors through stable tokens and dark overrides. Tests assert that the theme contract remains present.

**Tech Stack:** Astro 7 static site, vanilla JavaScript, CSS custom properties, Node source assertions.

### Task 2: Layout Theme Bootstrap and Header Button

**Files:**
- Modify: `src/layouts/SiteLayout.astro`

- [ ] **Step 1: Add early theme boot script before the stylesheet**

Add an inline script in `<head>` that reads `localStorage.getItem("toolman-theme")`, falls back to `window.matchMedia("(prefers-color-scheme: dark)")`, and sets `document.documentElement.dataset.theme`.

- [ ] **Step 2: Add a fixed-size header theme toggle**

Add `<button class="theme-toggle" type="button" data-theme-toggle aria-label="切换为暗黑模式" aria-pressed="false"><span class="theme-toggle-icon" aria-hidden="true"></span></button>` inside `.header-actions`, after search submit.

- [ ] **Step 3: Bump asset version**

### Task 3: Theme Interaction Script

**Files:**
- Modify: `public/site.js`

- [ ] **Step 1: Add theme helpers**

Add helpers for `getSavedTheme`, `getSystemTheme`, `applyTheme`, and `syncThemeToggle` using the `toolman-theme` localStorage key.
